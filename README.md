### Задание:
спроектировать минималистичный REST API сервис для книжного магазина.

Книга представлена уникальным названием, автором и годом выпуска.
Пользователь API может:

- получить список книг или конкретную книгу
- добавить или удалить книгу
- изменить информацию о существующей книге

Обязательные требования:

- Сервис должен предоставлять REST API по HTTP
- Минималистичность сервиса
- Использовать python 3; фреймворк flask или aiohttp
- Данные можно держать в памяти или в БД - на выбор
- Конфигурация приложения (как минимум порт) должна задаваться переменными окружениями
- Придерживаться конвенций RESTful (JSON в запросах и ответах, коды HTTP)

Необязательные требования:

- использование конвенций 12-факторного приложения
- пакетирование приложения в питоновский пакет или докер-образ
- написание хотя бы одного интеграционного теста

На выходе должен получиться архив с исходным кодом и инструкцией запуска. Всё остальное на ваше усмотрение.

Целью задания является не получение готового к эксплуатации продукта, а изучение вашего представления о проектирования сервисов. 

Не стоит тратить на выполнение задания больше рабочего дня, особенно на необязательные требования.
